# iOS 无线投屏接收器设计

## 目标

基于 `serezhka/java-airplay` 构建一款个人使用的 Windows 便携投屏软件。软件接收 iPhone 和 iPad 的系统 AirPlay 屏幕镜像，在电脑上显示高清画面并播放系统声音。

最终交付为 ZIP 便携包：在 Windows 10/11 x64 电脑上解压后双击即可运行，不要求用户另行安装 Java、GStreamer 或 FFmpeg。

## 支持范围

- Windows 10 x64、Windows 11 x64。
- 首要测试设备为 iPhone 15（iOS 26）和 iPad mini（iPadOS 26）。
- 同一局域网内的无线 AirPlay 屏幕镜像。
- 单台 iPhone 或 iPad 同时连接。
- 最高 1920×1080、30fps，提供 1080p 和 720p 两档，默认 1080p。
- 接收并播放 H.264 视频与 ALAC/AAC-ELD 音频。
- 简洁窗口、全屏、窗口置顶、音量调节、接收器名称修改和主动断开。

首版不包含录屏、远程控制、互联网跨网络投屏、多设备同时显示、账号系统或自动更新。受 DRM/HDCP 保护的视频不属于兼容性承诺范围。

## 技术路线

首版保留并完善 `java-airplay` 的 AirPlay 协议实现，使用 GStreamer 作为唯一正式播放后端，并在其上增加 Windows 桌面界面和便携打包。

先完成协议验证，再进行产品化：必须先证明 iOS 26/iPadOS 26 可以发现接收器、建立连接并持续播放音视频，之后才完善界面和打包。这避免在底层协议不兼容时提前投入界面工作。

## 架构

系统由四部分组成：

1. **设备发现与 AirPlay 服务**：通过局域网服务发现让 Windows 电脑出现在 iOS/iPadOS“屏幕镜像”列表中，处理连接、配对、会话和媒体数据接收。
2. **媒体播放引擎**：使用 GStreamer 解码 H.264 视频以及 ALAC/AAC-ELD 音频，并将声音输出到 Windows 当前默认播放设备。
3. **Windows 桌面界面**：显示等待状态、连接状态和投屏画面，提供全屏、置顶、音量、接收器名称与断开控制。
4. **便携运行环境**：通过精简 Java 运行时和所需 GStreamer 组件组成自包含目录，配置与日志保存在程序目录，不依赖注册表保存用户设置。

模块边界保持清晰：协议层只产生会话状态和媒体流；播放层只负责解码与输出；界面通过稳定的状态接口控制服务和播放器；打包层不改变业务行为。

## 使用流程

1. 用户解压 ZIP 并双击启动程序。
2. 程序启动 AirPlay 服务，窗口显示“等待 iPhone 或 iPad 连接”和当前接收器名称。
3. 首次运行如需 Windows 防火墙放行，程序触发系统授权提示并在失败时显示处理说明。
4. 用户在 iPhone/iPad 控制中心打开“屏幕镜像”并选择电脑。
5. 握手成功后，程序切换到播放界面并同时播放画面和声音。
6. 设备横竖屏切换时，画面保持原始比例并自动适配窗口，不进行拉伸。
7. 用户从移动设备停止镜像或在电脑端主动断开后，程序释放本次会话并返回等待页面。
8. 后续连接不要求重启程序。

首版不设置 AirPlay 密码，仅接受同一局域网中的连接。

## 状态与异常处理

界面至少区分等待、连接中、播放中、短暂重连和错误五种状态。

- **无法发现电脑**：提示检查同一局域网、Windows 专用网络设置、VPN、本地网络隔离和防火墙权限。
- **握手失败**：终止当前会话、保存兼容性日志并恢复等待状态，程序不得退出。
- **有画面无声音**：记录收到的音频格式、播放管线状态与 Windows 输出设备，界面显示音频异常。
- **有声音无画面**：记录视频格式与解码错误，界面显示视频异常。
- **网络波动**：短时中断进入重连状态；会话确认失效后释放资源并返回等待状态。
- **连续卡顿**：允许用户切换到 720p；质量切换通过重建会话或播放管线完成，不保留不确定的半连接状态。
- **异常退出**：保留最近一次结构化日志，下一次启动不自动恢复旧会话。

日志不得记录用户播放内容本身，只记录协议阶段、媒体格式、状态变化和错误信息。

## 界面

首版采用单窗口简洁界面：

- 等待页：软件名称、当前接收器名称、等待提示和简短连接指引。
- 播放页：视频区域及可自动隐藏的底部控制栏。
- 控制项：全屏、置顶、音量、1080p/720p、断开。
- 设置项：接收器名称和日志目录入口。

界面不承担复杂设备管理，也不加入账号、商店、广告或云服务。

## 便携打包

便携包包含应用程序、精简 Java 运行时、GStreamer 必需组件、默认配置、许可证及启动入口。目标电脑不需要管理员权限完成日常启动；防火墙规则若需要提升权限，则交由 Windows 系统提示用户确认。

便携包允许放置在不同目录，但程序目录需要可写，以保存个人配置和日志。完整 ZIP 体积可能达到数百 MB，这是自包含运行环境的预期代价。

## 验证策略

### 自动验证

- 协议消息解析和关键会话状态转换。
- 配置的默认值、保存、加载和非法值回退。
- 连接、断开、重复连接及错误恢复状态。
- 界面控制与播放器/服务命令的边界行为。

### 本机集成验证

- 自包含运行时与 GStreamer 组件加载。
- AirPlay 服务发布和局域网发现。
- 防火墙提示与错误说明。
- Windows 音频输出、全屏、置顶、音量和横竖屏切换。

### 真实设备验证

- iPhone 15（iOS 26）和 iPad mini（iPadOS 26）能够发现、连接、断开和再次连接。
- 普通系统界面、相册、本地非加密视频、网页及常见非 DRM 应用能够投屏。
- 画面与系统声音同时在电脑播放。
- 连续投屏至少 30 分钟不崩溃，音画不同步不随时间持续累积到明显程度。

### 便携性验证

在未单独安装项目运行依赖的 Windows 10 x64 和 Windows 11 x64 电脑上，分别验证解压、启动、发现、连接和播放。

## 验收标准

- ZIP 解压后可通过单一启动入口运行。
- 不需要用户另行安装 Java、GStreamer 或 FFmpeg。
- Windows 10/11 x64 均可运行。
- 指定 iPhone 和 iPad 可发现接收器并重复建立连接。
- 普通非 DRM 内容最高支持 1920×1080、30fps，且没有使用时长或高清时长限制。
- 普通视频的画面和系统声音均从电脑输出。
- 横竖屏切换比例正确。
- 连续运行 30 分钟无崩溃和持续累积的明显音画不同步。
- 全屏、置顶、音量、质量档位、接收器名称及断开功能可用。
- 错误不会导致服务进程无提示退出，并能留下足够但不包含媒体内容的诊断日志。

## 已知风险

- 原项目自 2023 年后未积极维护，iOS 26/iPadOS 26 的握手、加密或媒体格式可能需要适配。
- GStreamer 的 Java 集成、原生库加载与便携打包可能是 Windows 部署的主要难点。
- 当前上游存在音频映射失败和音频导致视频停止等未解决问题，音画并行播放必须通过真实设备验证。
- DRM/HDCP 内容可能黑屏、静音或拒绝投屏，首版不绕过内容保护。
